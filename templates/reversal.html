Transaction Reversals

Compensate or cancel a prior transaction event. The loyalty engine calculates previously issued points and deducts them via a ledger reversal entry.

↩️ Process Reversal

Enter the exact Event ID that was processed originally.

⚠️ Reversal Rules

Immutable Ledger Principle: The database does not delete rows. Instead, it writes a compensating entry with negative points.

Idempotent Safety: An event can only be reversed once. Re-applying a reversal on the same Event ID will be rejected by the engine.

Insufficient Balance: If reversing an event drives a user's points balance below zero, the reversal is still processed, but their balance will show as negative.
